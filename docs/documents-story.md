As a user, I want to see a list of my saved legal documents so that I can browse, filter, and access them.

## Requirements

- Display a page title "Documents" as H1.
- Show saved documents in a responsive card grid layout using `grid-template-columns: repeat(auto-fill, minmax(300px, 1fr))`.
- Each document card displays: title, status badge (Done/Draft/Generating), 2-line description snippet, type badge, date, and page count.
- Status badges use color variants: success for Done, warning for Draft, primary for Generating.
- Provide a sort dropdown with "Most Recent" and "A-Z" options.
- Provide a type filter dropdown with "All Types", "Contract", "Policy", "Employment", and "NDA" options.
- Include a persistent left sidebar (240px) with "Documents" (active) and "New Document" navigation items.
- Include a top bar (56px) with search input (280px max-width), and user avatar. Search is placeholder-only for this story — visible but non-functional.
- Responsive: at <768px, sidebar collapses behind a hamburger menu with overlay; card grid collapses to 1 column; content padding reduces to 16px.
- Responsive: at 768px-1023px, card grid shows 2 columns.
- Each card is clickable, navigating to the document detail view.
- Cards show hover state with shadow and stronger border.
- Show a loading state with 6 skeleton cards (animate-pulse blocks mirroring card anatomy) while documents are being fetched. Sort and type filter dropdowns are visible but disabled during loading.
- Show an empty state when no documents exist: centered message "No documents yet." / "Create your first document to get started." with a "New Document" CTA button. When caused by a filter returning zero results, show "No documents match your filters." / "Try adjusting your filters." with no CTA button.
- Below the card grid, show a centered "Load more" outline button that fetches the next page of results. While fetching, the button label changes to a spinner and the button is disabled. The button is hidden when all documents have been loaded.
- When any API fetch returns an error response, send it to a global dialog store which determines the dialog title, description, and variant to display. Render a global dialog component (using shadcn's dialog) that reads from the store and shows/hides accordingly. The dialog includes a close button that clears the store.
- All user-facing strings are externalised through i18next.
