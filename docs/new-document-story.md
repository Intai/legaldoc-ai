As a user, I want to create a new legal document through a guided 3-step flow — select references, provide context, and review the generated result — so that I can produce tailored legal documents grounded in my own reference materials.

## Requirements

- Display the New Document page at route `/documents/new` with a 3-step wizard: Select References, Provide Context, and Review & Save.
- Step indicator at the top shows all three steps with active/complete/upcoming states. Completed steps show a checkmark icon in a filled primary-700 circle. Active step shows the step number in a primary-100 circle with primary-400 border. Upcoming steps show the number in a neutral-100 circle. Connectors between steps use primary-300 when the preceding step is complete, default border color otherwise. Clicking a completed step's circle navigates back to that step.
- On mobile (<768px), step indicator labels are hidden, showing only numbered/checkmark circles with connectors.
- **Step 1 — Select References:**
  - Upload area at the top using `react-dropzone` with dashed border (2px, neutral-300), centered upload icon, primary text "Drag & drop files here, or click to browse", and helper text "PDF, TXT supported". Accepts only `application/pdf` and `text/plain` MIME types. On drag-over: border color changes to primary-400 with primary-50 background tint. Dropped files are uploaded via `POST /v1/references` and appear in the reference list immediately upon success.
  - Below upload: a search input and type filter dropdown ("All Types", "Contract", "Policy", "Employment") in a row.
  - Two-column layout below filters: left column is a scrollable reference list (`flex: 1`), right column is a selected references panel (fixed 280px width), 16px gap between columns.
  - Reference list items show: checkbox, title (16px semibold), 1-line description snippet (14px), badge + date (12px meta row). Checked items get primary-50 background. Items separated by 1px neutral-100 border-bottom.
  - Selected panel: header "Selected (N)" with count, lists selected reference titles with type badges. Each item has a permanently visible remove (x) button that highlights on hover (error-600 text, error-50 background). Show "No references selected" when empty.
  - On mobile: two-column layout stacks vertically, selected panel is hidden and replaced by a collapsible summary bar above the reference list showing "N references selected".
  - Footer bar: sticky at the bottom, border-top. Back button (secondary, disabled on step 1). Next button (primary, right-aligned, disabled until at least 1 reference selected).
  - Reference list is populated from `GET /v1/references` API endpoint.
  - Search filters references client-side by title and description. Type filter filters by document type.
- **Step 2 — Provide Context:**
  - Selected references summary card at top (neutral-50 background, border, 12px radius) showing count and reference titles with type badges.
  - Label "Describe what you need" (14px, semibold, neutral-700) with helper text "Include the parties, purpose, specific terms, and any questions you want addressed."
  - Textarea: full width, minimum height 200px, resizable vertically, with focus ring.
  - Footer bar: Back button (secondary, enabled, navigates to step 1). Generate Document button (primary, disabled if textarea is empty).
  - On mobile: Back and Generate Document sit side-by-side.
- **Step 3 — Review & Save:**
  - Phase progress indicator showing four phases: "Analyzing references", "Structuring document", "Drafting sections", "Finalizing". Each phase has an icon: completed (checkmark in success-700), active (spinner in primary-500), pending (empty circle in neutral-400). Container has neutral-50 background, border, 8px radius, max-width 720px centered.
  - Document viewer below phase progress renders the generated PDF using `react-pdf`, reusing the same rendering approach as the document detail page. Max-width 720px, centered. The PDF is fetched via `GET /v1/documents/{id}/pdf` once generation completes.
  - During generation, Save/Export buttons are disabled.
  - Mock the AI generation: `POST /v1/documents/generate` accepts `referenceIds` and `context`, creates a document with status `Draft` in MongoDB, generates a PDF using `reportlab` with hardcoded NDA content matching the UI design, stores the PDF in the document's `pdf_content` field, and returns phase progress via SSE. The SSE stream emits phase events (`analyzing`, `structuring`, `drafting`, `finalizing`) with simulated delays, and a final `complete` event containing the document ID.
  - Footer bar: Back button (secondary, navigates to step 2). Save button (primary). Export PDF button (outline style). On mobile: Export PDF becomes icon-only (download icon).
  - Save button calls `PUT /v1/documents/{id}/status` to update the document status from `Draft` to `Done` and navigates to the document detail page on success.
  - Export PDF triggers a browser download via the shared `downloadFile` utility using `GET /v1/documents/{id}/pdf`. The server provides the filename via `Content-Disposition` header.
- Sidebar "New Document" navigation item is active on this page.
- All user-facing strings are externalised through i18next.
- Follow the layout from `docs/new-document-layout.md` and styling from `docs/new-document-ui-design.html`.
